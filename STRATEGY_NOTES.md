# Storvex Strategy Notes

## Organic Compounding Value — Key Selling Point

**Core thesis:** Storvex gets organically stronger with every deal that flows through it.

### How it compounds:

1. **Every REC Package creates training data.** Each recommendation captures the full analytical context — SiteScore dimensions, market demographics, pricing model outputs, and the deal verdict. Over time, this builds a proprietary dataset of what PS actually greenlit vs. passed on, and why.

2. **SiteScore weights self-calibrate.** The 11-dimension scoring model (population, growth, income, zoning, PS proximity, etc.) starts with informed assumptions. As approved deals move through construction and lease-up, actual performance data can validate or challenge the weights — shifting from expert opinion to empirically-backed scoring.

3. **Pricing model accuracy improves.** Static assumptions today (3% annual escalation, fixed OpEx ratios, cap rate bands) get replaced with market-calibrated inputs as more deals close and stabilize. The gap between projected NOI and actual NOI narrows with each completed project.

4. **Pattern recognition across markets.** As REC Packages accumulate across Southwest, East, and future regions, Storvex surfaces cross-market patterns — which demographic profiles consistently produce elite-performing facilities, which zoning environments correlate with faster entitlement timelines, which competitive densities still support strong lease-up.

5. **Speed advantage widens.** Each deal processed refines the workflow. Institutional-quality underwriting that takes competitors weeks happens in hours. In competitive land markets, speed-to-LOI is the edge — and that edge grows as the system learns.

### The flywheel:

More deals through the platform -> more outcome data -> better scoring & pricing -> better deal selection -> higher hit rate -> more deals through the platform.

### Strategic implication:

Storvex isn't a static tool — it's a compounding asset. Every REC Package makes the next one more accurate. Competitors who start later can't shortcut the dataset advantage. This is the defensible moat.
